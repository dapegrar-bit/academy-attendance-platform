# منصة أكاديميتي لإدارة الحضور والغياب والمحاضرات

نظام ويب عربي كامل لإدارة تحضير المتدربين في الأكاديمية. المتدرب لا يرى رابط Zoom إلا بعد الضغط على زر التحضير، وتظهر للإدارة بيانات الحضور كاملة: وقت التسجيل، الرابط الذي ظهر للمتدرب، بياناته، الدفعة، المحاضرة، IP، والتقارير.

## التقنيات

- Python 3.12
- Django 5.2+
- SQLite للتجربة المحلية
- HTML / CSS / JavaScript بدون قوالب خارجية
- دعم RTL كامل للغة العربية
- جاهز للرفع على GitHub والنشر لاحقًا

## المجلدات المهمة

```text
academy_attendance_platform/
├── academy_platform/          # إعدادات مشروع Django والروابط العامة
├── attendance/                # تطبيق الحضور والمحاضرات
│   ├── models.py              # الجداول: البرامج، الدفعات، المحاضرات، الحضور، المتدربين
│   ├── views.py               # منطق الصفحات ولوحات التحكم
│   ├── forms.py               # النماذج
│   ├── admin.py               # إدارة Django Admin
│   └── management/commands/   # أوامر تجريبية مثل seed_demo
├── templates/attendance/      # قوالب HTML العربية
├── static/css/style.css       # التصميم والألوان
├── static/js/main.js          # الرسم البياني البسيط
├── static/img/logo.png        # الشعار الشفاف القابل للاستبدال
├── docs/FEATURES.md           # وصف المميزات
├── Dockerfile                 # تشغيل عبر Docker
├── docker-compose.yml         # تشغيل محلي سريع عبر Docker Compose

```

## التشغيل المحلي بدون Docker

### 1) إنشاء بيئة افتراضية

```bash
python -m venv .venv
```

### 2) تفعيل البيئة

Windows:

```bash
.venv\Scripts\activate
```

macOS / Linux:

```bash
source .venv/bin/activate
```

### 3) تثبيت المتطلبات

```bash
pip install -r requirements.txt
```

### 4) إنشاء ملف البيئة

انسخ ملف `.env.example` إلى `.env`:

```bash
cp .env.example .env
```

في Windows يمكن استخدام:

```bash
copy .env.example .env
```

### 5) تطبيق قاعدة البيانات

```bash
python manage.py migrate
```

### 6) إنشاء بيانات تجريبية

```bash
python manage.py seed_demo
```

سيتم إنشاء:

- حساب الإدارة: `admin`
- كلمة المرور: `admin123`
- حساب المتدرب: `sara`
- كلمة المرور: `1234`

### 7) تشغيل الموقع

```bash
python manage.py runserver
```

افتح المتصفح على:

```text
http://127.0.0.1:8000/
```

## التشغيل عبر Docker

```bash
docker compose up --build
```

ثم افتح:

```text
http://127.0.0.1:8000/
```

## طريقة الاستخدام داخل المنصة

### أولًا: الإدارة

1. سجلي الدخول بحساب الإدارة.
2. أضيفي برنامج تدريبي.
3. أضيفي دفعة تدريبية مرتبطة بالبرنامج.
4. أضيفي المتدربين يدويًا أو اتركي لهم التسجيل من صفحة إنشاء الحساب.
5. أضيفي محاضرة يومية مع رابط Zoom.
6. راقبي الحضور من لوحة الإدارة أو من تقرير الحضور.
7. استخدمي التصدير CSV عند الحاجة.

### ثانيًا: المتدرب

1. يسجل الدخول باسم المستخدم وكلمة المرور.
2. تظهر له محاضرات اليوم حسب الدفعة.
3. يضغط زر: تحضير وفتح زووم.
4. يتم حفظ وقت التحضير وبيانات الدخول.
5. يظهر له رابط Zoom بعد التحضير فقط.
6. تظهر نسبة حضوره في لوحة المتدرب.

## تغيير الشعار والألوان

- استبدلي الملف: `static/img/logo.png` بالشعار الشفاف الخاص بالأكاديمية.
- لتغيير الألوان افتحي: `static/css/style.css` وعدلي المتغيرات أعلى الملف:

```css
:root {
  --primary: #0b6b72;
  --primary-dark: #084a55;
  --accent: #d8a847;
}
```

## ملاحظات أمان قبل النشر الحقيقي

- غيّري `SECRET_KEY` في ملف `.env`.
- اجعلي `DEBUG=False` في الإنتاج.
- ضعي دومين الموقع داخل `ALLOWED_HOSTS`.
- ضعي رابط الموقع داخل `CSRF_TRUSTED_ORIGINS`.
- يفضل استخدام PostgreSQL بدل SQLite في الإنتاج.
- يفضل تفعيل HTTPS من الاستضافة.

## مميزات جاهزة في النسخة الحالية

- تسجيل دخول الإدارة والمتدربين.
- تسجيل ذاتي للمتدربين أو إضافة من الإدارة.
- برامج تدريبية ودفعات.
- روابط Zoom حسب المحاضرة والدفعة.
- روابط Zoom مخصصة لمتدرب معين عبر Django Admin.
- زر تحضير قبل إظهار الرابط.
- حفظ وقت التحضير، IP، المتصفح، والرابط الذي ظهر للمتدرب.
- لوحة إدارة بإحصائيات ورسم بياني.
- لوحة متدرب بنسبة حضور مباشرة.
- فلاتر حسب التاريخ والدفعة والحالة والاسم/الهوية.
- تصدير CSV.
- إعلانات عامة أو مخصصة للمتدربين/الإدارة.
- تصميم عربي احترافي متجاوب.

- بطاقات إحصائية داكنة، جداول خضراء، أزرار عريضة، ونوافذ منبثقة للدفعات.
